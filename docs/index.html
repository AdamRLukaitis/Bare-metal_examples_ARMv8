Examples - ARM®DS-5™

DS-5 provides a selection of examples to help you get started. The examples are provided in one or more archive files. See installation instructions for information about installing the examples. The following examples are available:

ARMv8 Bare-Metal

Bare-metal software examples for ARMv8 that illustrate compilation with ARM Compiler 6 and bare-metal debug with DS-5 Debugger. The examples are located in the archive file <examples_directory>\Bare-metal_examples_v8.zip.

  • Bare-metal ARMv8 Startup Code example for ARM Compiler 6 and the Foundation Platform model

    Bare-metal ARMv8 Startup Code example to demonstrate support for building and debugging ARMv8 bare-metal applications with ARM Compiler 6 and DS-5 Debugger.

  • Calendar Bare-metal Example for ARMv8 and ARM Compiler 6

    This example illustrates debugging a simple bare-metal calendar application that contains a logic error. Note: A 64-bit installation of DS-5 with an Ultimate Edition license are required to make full use of this example.

  • Fireworks Bare-metal ARMv8 Example for ARM Compiler 6 and the Foundation Platform model

    Fireworks example to demonstrate support for building and debugging ARMv8 bare-metal applications with ARM Compiler 6 and DS-5 Debugger.

  • Fireworks Bare-metal ARMv8 SMP Example for ARM Compiler 6 and the AEMv8 FVP model

    Fireworks example to demonstrate support for building and debugging ARMv8 SMP bare-metal applications with ARM Compiler 6 and DS-5 Debugger. Note: A 64-bit installation of DS-5 with an Ultimate Edition license are required to make full use of this example.

  • Startup Code Example for ARMv8 AArch64 (with AArch32 application) on the AEMv8 FVP model

    This is a bare-metal SMP Startup Code example for ARMv8 that initializes the system in AArch64 EL3 then launches an AArch32 application in EL1. Note: A 64-bit installation of DS-5 with an Ultimate Edition license are required to make full use of this example.

  • Startup Code Example for ARMv8 AArch64 on the AEMv8 FVP model

    This is a bare-metal Startup Code example for ARMv8 AArch64 written in C and assembler that runs on the AEMv8 FVP model. Note: A 64-bit installation of DS-5 with an Ultimate Edition license are required to make full use of this example.

  • Startup Code Example for Cortex-A53, Cortex-A57 and Cortex-A72 on the ARM Juno and ARM LogicTile Express platforms

    This is a bare-metal Startup Code example for Cortex-A53, Cortex-A57 and Cortex-A72, written in C and assembler that runs on the ARM Juno and ARM LogicTile Express platforms. Note: A 64-bit installation of DS-5 with an Ultimate Edition license are required to make full use of this example.

ARMv7 Bare-Metal

Bare-metal software examples for ARMv7 (and earlier) that illustrate compilation with ARM Compiler 5 and bare-metal debug with DS-5 Debugger. The examples are located in the archive file <examples_directory>\Bare-metal_examples_ARMv7.zip.

  • ARM Compiler 5 Optimization 1 - Basics

    This example introduces some basic code size and performance optimization and measurement techniques for application software using ARM Compiler 5, and is illustrated by a simple bare-metal semihosted "sorts" example application.

  • ARM Compiler 5 Optimization 2 - Additional Techniques

    This example introduces some additional optimization techniques for application software using ARM Compiler 5, including multi-file optimization, linker feedback, floating point and NEON auto-vectorization. It also describes some ways to estimate and reduce Stack and Heap usage.

  • ARM Compiler 5 Optimization 3 - NEON auto-vectorization of bare-metal Fireworks example

    This example demonstrates NEON auto-vectorization of the bare-metal Fireworks example using ARM Compiler 5, and shows ways of modifying parts of the source code to take better advantage of NEON auto-vectorization, and ways of measuring the performance gains of NEON auto-vectorization.

Debug and Trace Services Layer (DTSL)

Software examples for DS-5 Debugger's Debug and Trace Services Layer (DTSL). The examples are located in the archive file <examples_directory>\DTSL_examples.zip.

  • DTSL Example Configdb Example

    This directory contains the DS-5 configdb entries used/referenced by the DTSL examples.

  • DTSL Java Example

    This is an example of how one can use DTSL in 'stand-alone' mode (i.e. outside of the Eclipse environment). DTSL is mainly written in Java and so can be used by either a Java program, as in this example, or by a Jython (Python for Java) program.

  • DTSL Jython Example

    This is an example of how one can use DTSL in 'stand-alone' mode (i.e. outside of the Eclipse environment). DTSL is mainly written in Java and so can be used by either a Java program or, as in this example, by a Jython (Python for Java) program.

  • DTSL Jython JTAG SVF Example

    This is an example of how one can use DSTREAM to process Serial Vector Format (SVF) files and SVF statements. This uses DTSL to create a DSTREAM JTAG connection which is then used to interact with the JTAG SVF server on DSTREAM.

  • DTSL Jython Show Registers Example

    This example stand alone DTSL program will connect to a platform and show you the set of available registers for one device within the platform. What makes this example slightly more interesting than it first appears, is that because the example is simple it can attach to many platform types - including a models connection.

  • DTSL Probe Vcc Example

    This is an example of a stand alone DTSL program which connects to a DSTREAM unit and shows its configuration items and their current values. Some of these configuration items allow the identification of the probe connection currently in use along with the state of target Vcc.

  • DTSL STM Decode Example

    This is an example of how to write your own STM processing scripts to decode the STM data streams, and to display Master IDs, Channel IDs and the data content

  • DTSL STM File Decode Example

    This example shows how to decode STM trace data which has been captured into a file. The file data can be a DSTREAM buffer, an ETB buffer or just a raw STPv2 protocol stream.

  • DTSL Trace Dump Example

    This is an example of how you can dump a DTSL trace capture device's buffer to a disc file. The example has been written so that it may be used both as a stand alone DTSL program, or as a DS-5 Debugger script.

  • DTSL Trace Stats Example

    This is an example of how one can process trace data using DTSL. This uses DTSL to create a trace processing pipeline which identifies all ATB ID sources within the trace capture device. This example can be run either stand alone or within a DS-5 debug session.

Jython

Jython examples for DS-5 Debugger. The examples are located in the archive file <examples_directory>\Jython_examples.zip.

  • Jython MD5 Hash Example

    This example illustrates the use of a Jython script in DS-5 Debugger to generate an MD5 Hash on some data in memory.

  • Jython Performance Monitor Unit (PMU) Example

    This example illustrates the use of a Jython script in DS-5 Debugger to use the Performance Monitor Unit (PMU) of Cortex-A15 to count elapsed cycles, and user-defined events, between two points in program execution.

  • Jython State Display Example

    This example illustrates the use of a Jython script in DS-5 Debugger to display the state (stack frame / execution context) of an application.

  • Jython Translation Table Decoder Example

    This example illustrates the use of a Jython script in DS-5 Debugger to decode and display the contents of the Translation Tables used by the Memory Management Unit (MMU) in a target.
