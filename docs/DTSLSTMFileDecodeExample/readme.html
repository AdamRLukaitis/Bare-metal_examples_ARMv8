DTSL STM File Decode Example - ARM®DS-5™

Table of Contents:

This example shows how to decode STM trace data which has been captured into a file. The file data can be a DSTREAM buffer, an ETB buffer or just a raw STPv2 protocol stream.
Note
For information on locating the example files and extracting them for use either on the command-line or in Eclipse, see the installation instructions.

Purpose and scope

The Debug and Trace Services Layer (DTSL) is a software layer that sits between DS-5 Debugger and the RDDI target access API. DS-5 Debugger uses DTSL to create target connections, configure the target platform to be ready for debug operations, and communicate with the debug components on the target.
As a power user of DS-5 Debugger, you might need to use DTSL as part of new platform support, to extend the capabilities of DS-5 Debugger, to add support for custom debug components, or to create your own Java or Jython programs which interact with your target. These examples illustrate these uses.

Getting Started with the DTSL Examples

The DTSL examples are provided in [DS-5 install dir]\examples\DTSL_examples.zip. To access these, either import the examples into the DS-5 Eclipse workspace, or unzip them to a suitable location.
Some additional files are provided in the [DS-5 install dir]\sw\DTSL folder. To use the examples you must either import the DTSL library DTSL.zip from this folder into the DS-5 Eclipse workspace, or unzip it to a suitable location. This folder also contains the DTSL Java docs converted into a PDF document DTSLJavaDocs.pdf and the zipped up content of the DTSL Javadocs com.arm.debug.dtsl.docs.zip.
If you use the Cheat Sheet to "Automatically import the DS-5 Example Projects into the current workspace", both the DTSL examples [DS-5 install dir]\examples\DTSL_examples.zip and the DTSL library [DS-5 install dir]\sw\DTSL\DTSL.zip will be imported automatically for you into the DS-5 Eclipse workspace.
See the readme.txt text files within each sub-directory for further information.
